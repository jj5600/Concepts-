week0 :
Before STM32CubeIDE 2.0
The current videos and slides do this with 1.x version of STM32CubeIDE:

Open project in STM32CubeIDE
Double-click .ioc
Configure pins/clocks/peripherals inside the IDE
Click Generate Code
Keep coding/debugging in the same tool
With STM32CubeIDE 2.0
Now you do this instead:

Open/edit .ioc in standalone STM32CubeMX
Configure pins/clocks/peripherals there
Generate code/project
Open/import that project into STM32CubeIDE
Do your coding, build, flashing, debugging in CubeIDE
